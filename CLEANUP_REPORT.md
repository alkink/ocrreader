# Cleanup Report

This report lists files and directories that look unnecessary to keep under version control for the current state of this project.

I focused on items that are:
- generated automatically,
- local-machine specific,
- debug / profiling artifacts,
- benchmark output snapshots,
- temporary session logs.

I did **not** classify source code, config files, test scripts, documentation, or the main input dataset as unnecessary.

## Current Scan Summary

At the time of the scan, the repository contained at least:

- `output/`: about **74** generated files
- `tests/` text/json artifacts: about **31** files
- `dataset/generated/qa/` benchmark/orientation output files: about **503** files
- root session logs: **3** files
- Python cache files under `__pycache__/`: about **61** files

Also note:
- this folder is **not currently initialized as a Git repository**, so `.gitignore` has been prepared for future Git usage.

## Safe To Ignore

These are the files/directories I consider safe to keep out of Git by default.

### 1) Local Python cache and environments

- `__pycache__/`
- `*.py[cod]`
- `.pytest_cache/`
- `.mypy_cache/`
- `.ruff_cache/`
- `.venv/`
- `venv/`
- `.wsl-glm-venv/`

Why:
- machine-local,
- reproducible,
- not part of the project source.

### 2) OCR outputs and debug overlays

- `output/`

Why:
- generated by pipeline runs,
- large and noisy,
- easy to recreate.

Examples currently present:
- `output/debug/`
- `output/debug_v2/` ... `output/debug_v12/`
- `output/testdata_v31/`

### 3) Test / profiling artifacts

- `tests/*.txt`
- `tests/*.json`
- `tests/*.log`
- `tests/batch_user_image_results/`
- `tests/profile_paddle_vl_roi_outputs/`

Why:
- these are output dumps, inspection snapshots, and profile results,
- the actual useful test scripts are the `.py` files, not these generated result files.

Examples currently present:
- `tests/user_image_full_res.txt`
- `tests/vl_raw_res.json`
- `tests/profile_paddle_vl_baseline.json`
- `tests/full_vl_result.txt`

### 4) Generated QA / benchmark snapshots

- `dataset/generated/qa/bench*/`
- `dataset/generated/qa/benchmark*/`
- `dataset/generated/qa/orientation_audit*/`
- `dataset/generated/qa/anchor_label_probe.csv`
- `dataset/generated/qa/conflict_report.csv`
- `dataset/generated/qa/issues.csv`
- `dataset/generated/qa/coverage.csv`
- `dataset/generated/qa/summary.json`
- `dataset/generated/qa/glm_analysis.txt`

Why:
- these are experiment outputs,
- they are numerous,
- they change frequently,
- they can be regenerated from scripts.

### 5) Local task/session dumps

- `roo_task_*.md`
- `roo_task_*.txt`

Why:
- they are conversation/session dumps,
- very large,
- not part of the runnable application.

Examples currently present:
- `roo_task_mar-18-2026_9-33-02-am.md`
- `roo_task_mar-23-2026_3-30-35-pm.md`

### 6) OS noise

- `.DS_Store`
- `Thumbs.db`
